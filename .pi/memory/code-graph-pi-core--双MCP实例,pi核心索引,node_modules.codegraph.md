# code-graph-pi-core 双 MCP 实例

关键词：`code-graph` `pi-core` `node_modules` `双实例` `MCP` `索引`

## 问题

pi 核心代码安装在 `node_modules/@earendil-works/pi-coding-agent/dist/` 下，code-graph 用 Rust 的 `ignore` crate **硬编码排除 node_modules**，且**不跟随 symlink**。导致项目级 code-graph 无法索引 pi 核心 API（如 ExtensionAPI、事件类型等）。

## 排除的方案

| 方案 | 原因 |
|------|------|
| symlink 到项目目录 | code-graph 不跟随 symlink（已验证） |
| bind mount | 需要 sudo，重启失效 |
| 复制目录 | 占磁盘，需同步 |
| 修改 code-graph 源码 | 预编译 Rust 二进制，改不了 |

## 采用方案：双 MCP 实例

在 `~/.pi/agent/mcp.json` 注册第二个 code-graph 实例，CWD 指向 pi 核心安装目录。

### 配置

```json
{
  "code-graph-pi-core": {
    "command": "bash",
    "args": ["-c", "cd <pi-core-dir> && exec code-graph serve"],
    "lifecycle": "lazy"
  }
}
```

### pi-core 目录初始化

code-graph 要求目标目录有 `.git` 或已有 `.code-graph/`：

```bash
cd /home/lain/.local/share/fnm/node-versions/v22.22.2/installation/lib/node_modules/@earendil-works/pi-coding-agent
git init
echo -e "node_modules/\n.code-graph/" > .gitignore
code-graph incremental-index
```

### 工具名

pi 的 mcp-lite 用 `{serverName}_{toolName}` 格式注册工具：
- 项目级：`code_graph_project_map`、`code_graph_get_ast_node` 等
- pi 核心：`code_graph_pi_core_project_map`、`code_graph_pi_core_get_ast_node` 等

### 索引统计

- 443 文件 / 5530 nodes / 6463 edges
- 142 个 `.d.ts` + 145 个 `.js`（编译后代码，非 TS 源码）
- 搜索 `ExtensionAPI` 可正常返回符号定义

### 使用场景

当需要查看 pi 核心 API 签名、事件类型、扩展接口时：

```
code_graph_pi_core_get_ast_node(symbol_name="ExtensionAPI")
code_graph_pi_core_semantic_code_search(query="extension hook event types")
code_graph_pi_core_module_overview(path="dist/core/extensions")
```

## 维护注意事项

1. **pi 更新后**：npm 更新 pi 时可能删除 `.git` 和 `.code-graph/`，需重新初始化
2. **重新初始化脚本**：
   ```bash
   cd <pi-core-dir>
   [ -d .git ] || git init
   [ -f .gitignore ] || echo -e "node_modules/\n.code-graph/" > .gitignore
   code-graph incremental-index
   ```
3. **pi 版本变化**：fnm 切换 node 版本后路径会变，需要同步更新 `mcp.json` 中的路径
