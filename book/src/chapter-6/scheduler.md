# 6.2 pi-scheduler 原理：定时任务

> pi-scheduler 是 pi-atelier 的"闹钟"——它能在指定时间自动给 AI 注入消息，让 AI 主动执行任务。

## 为什么需要定时任务？

AI 是被动响应的——你问它才答。但有些事需要**按时做**：

- "30 分钟后提醒我看看 CI 结果"——你可能忘了
- "每 2 小时检查一下测试"——手动提醒太累
- "每天下班前提醒提交代码"——怕忘了

Scheduler 让 AI 有了"时间感知"能力。

## 工作原理

```
创建定时任务
     │
     ▼
┌──────────────────┐
│  Scheduler 定时器  │
│  倒计时等待触发    │
└────────┬─────────┘
         │ 到时间了！
         ▼
┌──────────────────┐
│  注入预设消息      │
│  到 AI 的上下文中  │
└────────┬─────────┘
         │
         ▼
     AI 读取消息
     自动执行任务
```

关键点：
- **注入消息**不是新建对话，而是在当前会话中插入一条"提醒"
- AI 看到消息后**自主决定**怎么执行，不需要你再说一遍
- 定时任务只在当前会话有效，会话结束自动清除

## 三种操作

### 创建任务

```
schedule(
  action="create",
  interval_ms=1800000,    // 30 分钟
  prompt="检查 CI 构建结果，如果失败告诉我",
  recurring=false          // 一次性
)
```

参数说明：

| 参数 | 说明 | 必填 |
|------|------|------|
| `action` | 固定 `"create"` | ✅ |
| `interval_ms` | 间隔毫秒 | ✅ |
| `prompt` | 注入给 AI 的消息 | ✅ |
| `recurring` | 是否重复（默认 false） | ❌ |

### 列出任务

```
schedule(action="list")

结果：
  📋 当前定时任务：
  1. [一次性] 14:30 触发 — "检查 CI 结果"
  2. [重复] 每 2h — "跑测试检查回归"
```

### 取消任务

```
schedule(action="cancel", id="task-123")
```

## 常用时间间隔

| 场景 | 间隔 | interval_ms |
|------|------|-------------|
| 短期提醒 | 5 分钟 | 300,000 |
| 泡杯茶的时间 | 15 分钟 | 900,000 |
| 等构建 | 30 分钟 | 1,800,000 |
| 定期检查 | 2 小时 | 7,200,000 |
| 每日提醒 | 1 天 | 86,400,000 |

## 一次性 vs 重复任务

### 一次性（recurring=false）

适合"提醒"场景：

```
用户："30 分钟后提醒我开会"

AI → schedule(action="create", interval_ms=1800000, 
              prompt="⏰ 提醒：该开会了。", recurring=false)

... 30 分钟后 ...

AI：⏰ 提醒：该开会了。当前时间 15:30。
```

触发一次后自动删除。

### 重复（recurring=true）

适合"定期检查"场景：

```
用户："每 2 小时跑一下测试"

AI → schedule(action="create", interval_ms=7200000,
              prompt="请运行 npm test，报告测试结果。如果失败，列出失败的测试。",
              recurring=true)

... 每 2 小时 ...

AI：📋 定期测试报告：所有 47 个测试通过 ✅
...
AI：⚠️ 定期测试报告：2 个测试失败！
    - auth.test.ts: 登录超时
    - api.test.ts: 404 错误
```

## 状态栏集成

Scheduler 在 pi 的状态栏显示倒计时，让你知道下一个提醒什么时候触发。

## 注意事项

- 定时任务**只在当前会话有效**——关闭会话后任务消失
- 重复任务不要间隔太短（建议 ≥ 5 分钟），避免频繁触发浪费 token
- prompt 要**具体明确**——AI 看到的是这段文字，模糊的指令会导致模糊的执行
